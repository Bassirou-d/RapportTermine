Les indicateurs

Premier Indicateur :

Énoncé

"Quels sont les produits conditionnés dans du carton ou dans du verre".

Interprétation

"Nous devons selectionner dans la base de donnes KDou tout les produits conditionnées en verre ou en carton".

Explication

Nous selectionnons les references,le nom,les quantites par unites de la table Produit,avec "LIKE" nous allons faire une recherche specifiée afin d'avoir toutes les quantites par unites contennant le mot "carton" ou "verre".

Ainsi nous obtiendrons tout les produits conditionnées en verre ou en carton.

Le Resultat :

Code SQL généré

SELECT Produit.RefProd, Produit.NomProd, Produit.QteParUnite
FROM Produit
WHERE ((Produit.QteParUnite) Like "*carton*" Or (Produit.QteParUnite) Like "*verre*");

Deuxieme Indicateur :

Énoncé

"Donner pour chaque fournisseur le nombre de produits qu'il propose en bouteille".

Interprétation

Ici nous devions selectionner le nombre de produit conditionnés en bouteille que chaque fournissseur propose.

Explication

Avec le "COUNT" nous comptons le nombre de contacts dans la table fournisseur et nous le renommons en "NombreDeBouteilles" grace a "AS".

Ensuite nous faions une jointure interne entre la table poduit et la table fournisseurs puis nous allons faire une recherche specifiée grace a "LIKE" nous selectionnerons tous les elements contennant le mot "bouteilles".

Pour finir nous regroupons les resultats par le contact du fournisseur et nous les trions de la meme maniere.

Le Resultat :

Code SQL généré

SELECT Fournisseur.Contact, COUNT(*) AS NombreDeBouteilles
FROM Fournisseur
INNER JOIN Produit ON Fournisseur.NoFour = Produit.NoFour
WHERE Produit.QteParUnite LIKE "*bouteilles*"
GROUP BY Fournisseur.Contact
ORDER BY Fournisseur.Contact;

Troisieme Indicateur :

Énoncé

"Est-ce qu’il y a dans la clientèle des clients qui n’ont pas encore passé de commande".

Interprétation

Ici nous devons regarder chez les clients,specifiquement ceux qui nont rien commandé.

Explication

Nous selectionnons les "Client" de la table "Client" et nous faisons une jointure gauche entre la table commande et la table client.

Grace au "WHERE" nous allons afficher les resultat ou le numero de commande et "null" c'est a dire ou il y a aucun element d'inscrit donc par consequent aucune commande passée.

Le Resultat :

Code SQL généré

SELECT Client.*
FROM Client
LEFT JOIN Commande ON Client.CodeCli = Commande.CodeCli
WHERE Commande.NoCom IS NULL;

Quatrieme Indicateur :

Énoncé

"Dans quelles villes la clientèle est-elle la plus importante ? Citer les 3 premières."

Interprétation

Ici nous devons cibler les villes avec la clientele la plus forte afin d'optimiser les operations economiques et marketing de l'entreprise KDou.

Explication

Nous selectionnons le top 3 des villes clients

Grace au "COUNT" nous les comptons et grace a "AS" nous les renommons en "NombreClientsparville".

Nous regroupons les resultats par ville et nous trions le comptage de maniere "DESC" c'est a dire de la valeur la plus élevée à la valeur la plus faible.

Le Resultat :

Code SQL généré

SELECT TOP 3 VilleCli, COUNT(*) AS NombreClientsparville
FROM Client
GROUP BY VilleCli
ORDER BY COUNT(*) DESC;
